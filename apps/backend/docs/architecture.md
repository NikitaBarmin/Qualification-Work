# Backend Architecture

## Что уже сделано
- Есть минимальный backend на `Express`.
- Сервер стартует через `index.ts`, а само приложение собирается в `src/app.ts`.
- Есть первый модуль `health`.
- Есть общие middleware для `404` и ошибок.
- Есть `env`-слой для переменных окружения.
- Есть `paths` и `storage`, чтобы backend заранее готовил рабочие директории.
- Подключён простой `SQLite core` с автоматическим созданием таблиц `users`, `datasets`, `analyses`.

## Как устроен запрос
1. `index.ts` запускает приложение.
2. `src/app.ts` собирает `Express app` и подключает middleware.
3. `src/routes.ts` подключает модули.
4. Роут модуля передаёт запрос в controller.
5. Controller работает с HTTP и вызывает service.
6. Service возвращает данные или бросает ошибку.
7. Если роут не найден, срабатывает `not-found`.
8. Если случилась ошибка, срабатывает `error-handler`.

## Почему это layered и при этом просто
- `routes` знают только URL и куда передать запрос.
- `controller` знает только HTTP-часть.
- `service` знает только логику и данные.
- `db` знает только базу данных.

Это уже layered-подход, но без лишних абстракций.

## Зачем нужны основные файлы
- `index.ts` — старт backend.
- `src/app.ts` — сборка `Express` приложения.
- `src/env.ts` — чтение и нормализация env.
- `src/routes.ts` — центральное подключение модулей.
- `src/middleware/not-found.ts` — ответ на неизвестный роут.
- `src/middleware/error-handler.ts` — единая обработка ошибок.
- `src/lib/app-error.ts` — общий класс ошибок приложения.
- `src/config/paths.ts` — все важные пути проекта.
- `src/config/storage.ts` — создание нужных директорий при старте.
- `src/db/sqlite.ts` — единая точка подключения к `SQLite`.
- `src/db/sqlite-schema.ts` — стартовая SQL-схема таблиц.
- `src/modules/health/*` — пример первого модуля.

## Что делает этап 2
- Backend заранее создаёт структуру хранения.
- Уже определены места для:
  - файла `SQLite`
  - временных загрузок
  - очищенных датасетов

Это помогает дальше спокойно подключать загрузку файлов и аналитику.

## Что делает этап 3
- Backend теперь реально поднимает `SQLite` при старте.
- Если файла базы ещё нет, он создаётся автоматически.
- Если таблиц ещё нет, они тоже создаются автоматически.
- `health` теперь показывает состояние `SQLite`, путь к файлу базы и список стартовых таблиц.

## Как это читать новичку
- Если нужно понять, как backend запускается, смотри `index.ts` и `src/app.ts`.
- Если нужно понять, куда идёт запрос, смотри `src/routes.ts` и нужный модуль в `src/modules`.
- Если нужно понять, где подключается база, смотри `src/db/sqlite.ts`.
- Если нужно понять, какие таблицы сейчас есть, смотри `src/db/sqlite-schema.ts`.

## Что будет дальше
- На следующем этапе поверх этого `SQLite core` можно спокойно добавить:
  - `auth`
  - `datasets`
  - `analyses`

То есть база уже не планируется, а реально существует как фундамент backend.
